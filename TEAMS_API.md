# Teams API Specification

Base URL: `{API_BASE_URI}` (configured via environment).

All endpoints expect/return `application/json`. Routes are reachable both with and without the `/api` prefix unless explicitly noted (e.g. `/teams` and `/api/teams` are equivalent).

This document covers the full Teams surface â€” CRUD plus the per-resource upload alias. The media-storage / presign contract used by `POST /teams/upload` is documented once in [`MEDIA_API.md`](./MEDIA_API.md) and is shared with the other per-resource upload aliases.

## Concepts

### Slug

Every team is identified by a URL-friendly `slug` (not a numeric id). The backend generates the slug, likely from `name`. The slug MUST be returned on every GET response and is used as the path parameter on `GET /teams/:slug`, `PUT /teams/:slug`, and `DELETE /teams/:slug`.

### Enums

| Field    | Values                                                                                                                                 | Default      |
|----------|----------------------------------------------------------------------------------------------------------------------------------------|--------------|
| `sport`  | `CRICKET \| RUGBY \| SOCCER \| BADMINTON \| BOWLING \| MOTOR_SPORTS \| BASKETBALL \| NETBALL \| VOLLEYBALL \| CYCLING \| OTHER`         | required     |
| `status` | `ACTIVE \| INACTIVE`                                                                                                                   | `ACTIVE`     |

### Relationship to other resources

- **Matches** (see [`MATCHES_API.md`](./MATCHES_API.md)) reference teams by free-text `homeTeamName` / `awayTeamName` strings, **not** by `Team.slug`. The backend MAY reconcile those names against existing `Team` records (matching by name within the team's sport) but is not required to enforce the FK.
- **Players** (see [`PLAYERS_API.md`](./PLAYERS_API.md)) carry a `team` field that is likewise a free-text team name, not a FK to `Team.slug`.

Renaming a `Team` therefore does NOT cascade into existing matches or players. If a rename happens, the backend SHOULD update referencing rows out-of-band.

### Authentication

- CRUD routes (`GET / POST / PUT / DELETE`) are **not** protected in code today, matching the rest of the resource spec.
- `POST /teams/upload` IS protected â€” send `x-access-token: <JWT>`. This mirrors all other `/<resource>/upload` aliases.

## Schema

### `Team`

```jsonc
{
  "slug":        "string",                                                       // canonical id, generated by backend
  "name":        "string",
  "sport":       "CRICKET | RUGBY | SOCCER | BADMINTON | BOWLING | MOTOR_SPORTS | BASKETBALL | NETBALL | VOLLEYBALL | CYCLING | OTHER",
  "status":      "ACTIVE | INACTIVE",
  "description": "string | null",
  "country":     "string | null",
  "city":        "string | null",
  "logoUrl":     "string | null",                                                // URL returned from /teams/upload (CDN / public URL)
  "createdAt":   "ISO8601",
  "updatedAt":   "ISO8601"
}
```

## Endpoints

### `GET /teams`

```jsonc
Response 200: Team[]
```

### `GET /teams/:slug`

```jsonc
Response 200: Team
Response 404: { "error": "Team not found" }
```

### `POST /teams`

```jsonc
Request:
{
  "name":        "string",                                                       // required
  "sport":       "CRICKET | RUGBY | SOCCER | BADMINTON | BOWLING | MOTOR_SPORTS | BASKETBALL | NETBALL | VOLLEYBALL | CYCLING | OTHER", // required
  "status":      "ACTIVE | INACTIVE",                                            // required, default ACTIVE
  "description": "string?",
  "country":     "string?",
  "city":        "string?",
  "logoUrl":     "string?"                                                       // set this to the `url` returned by /teams/upload
}

Response 201: Team
```

### `PUT /teams/:slug`

Same request shape as `POST`. Treat as a full replacement of the team document â€” the client always sends the complete state.

```jsonc
Response 200: Team
Response 404: { "error": "Team not found" }
```

### `DELETE /teams/:slug`

```jsonc
Response 204
```

## Logo upload

### `POST /teams/upload`

Protected. Returns presigned S3 upload details. Same response shape as `POST /media/presign` with the folder hard-coded to `teams` â€” the client SHOULD omit `folder` when calling the alias; the backend MUST ignore any client-supplied `folder` value and use `teams`.

See [`MEDIA_API.md`](./MEDIA_API.md) for:
- the canonical request / response payload,
- the browser â†’ S3 PUT flow (the API server never sees the file bytes),
- backend signing responsibilities (auth, key generation, size + content-type pinning, short expiry),
- and S3 bucket / CORS requirements.

Once S3 returns `2xx` on the `PUT`, the client stores the returned `url` in `Team.logoUrl` and submits the `POST /teams` or `PUT /teams/:slug` as usual.

## Error format

All non-2xx responses use the project-wide error shape:

```json
{ "error": "human-readable message", "code": "OPTIONAL_MACHINE_CODE" }
```
