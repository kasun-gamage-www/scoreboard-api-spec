# Other Events API Specification

Base URL: `{API_BASE_URI}` (configured via environment).

All endpoints expect/return `application/json`. Routes are reachable both with and without the `/api` prefix unless explicitly noted (e.g. `/other-events` and `/api/other-events` are equivalent).

This document covers the full Other Events surface — CRUD only. Other Events do not have a per-resource upload alias; if image attachments are added later they will follow the contract in [`MEDIA_API.md`](./MEDIA_API.md).

"Other Events" are calendar items that are not regular `Match` records — training sessions, exhibitions, ceremonies, press conferences, clinics, etc. They MAY be associated with a tournament, but generally stand alone.

---

## Concepts

### Slug

Every other event is identified by a URL-friendly `slug` (not a numeric id). The backend generates the slug, likely from `title`. The slug MUST be returned on every GET response and is used as the path parameter on `GET /other-events/:slug`, `PUT /other-events/:slug`, and `DELETE /other-events/:slug`.

### Enums

| Field        | Values                                                                                                                                          | Default       |
|--------------|-------------------------------------------------------------------------------------------------------------------------------------------------|---------------|
| `eventType`  | `TRAINING \| EXHIBITION \| CEREMONY \| AWARD_CEREMONY \| PRESS_CONFERENCE \| CLINIC \| OTHER`                                                    | required      |
| `status`     | `SCHEDULED \| LIVE \| COMPLETED \| CANCELLED \| POSTPONED`                                                                                      | `SCHEDULED`   |
| `sport`      | `CRICKET \| RUGBY \| SOCCER \| BADMINTON \| BOWLING \| MOTOR_SPORTS \| BASKETBALL \| NETBALL \| VOLLEYBALL \| CYCLING \| null`                   | nullable      |

The `status` enum is shared with `Match` (see [`MATCHES_API.md`](./MATCHES_API.md)). `sport` is nullable here because some events (e.g. an award ceremony covering multiple sports) are not tied to a single discipline.

### Relationship to other resources

- **Tournaments** (see [`TOURNAMENTS_API.md`](./TOURNAMENTS_API.md)) — `tournamentSlug` is a FK to `Tournament.slug`, or `null` when the event is standalone.
- **Venues** — `venue` is a free-text string (e.g. `"Dubai International Stadium"`), **not** a FK to `Venue.slug`. The backend MAY reconcile this against existing `Venue` records but is not required to enforce the relationship.

### Authentication

CRUD routes (`GET / POST / PUT / DELETE`) are **not** protected in code today, matching the rest of the resource spec.

## Schema

### `OtherEvent`

```jsonc
{
  "slug":           "string",                                                            // canonical id, generated by backend
  "title":          "string",
  "eventType":      "TRAINING | EXHIBITION | CEREMONY | AWARD_CEREMONY | PRESS_CONFERENCE | CLINIC | OTHER",
  "status":         "SCHEDULED | LIVE | COMPLETED | CANCELLED | POSTPONED",
  "sport":          "CRICKET | RUGBY | SOCCER | BADMINTON | BOWLING | MOTOR_SPORTS | BASKETBALL | NETBALL | VOLLEYBALL | CYCLING | null",
  "tournamentSlug": "string | null",                                                     // FK to Tournament.slug; null for standalone events
  "venue":          "string | null",                                                     // free-text venue name, not a FK
  "scheduledAt":    "ISO8601 | null",
  "organizer":      "string | null",
  "description":    "string | null",
  "createdAt":      "ISO8601",
  "updatedAt":      "ISO8601"
}
```

## Endpoints

### `GET /other-events`

```jsonc
Response 200: OtherEvent[]
```

### `GET /other-events/:slug`

```jsonc
Response 200: OtherEvent
Response 404: { "error": "Other event not found" }
```

### `POST /other-events`

```jsonc
Request:
{
  "title":          "string",                                                            // required
  "eventType":      "TRAINING | EXHIBITION | CEREMONY | AWARD_CEREMONY | PRESS_CONFERENCE | CLINIC | OTHER", // required
  "status":         "SCHEDULED | LIVE | COMPLETED | CANCELLED | POSTPONED",              // required, default SCHEDULED
  "sport":          "CRICKET | RUGBY | SOCCER | BADMINTON | BOWLING | MOTOR_SPORTS | BASKETBALL | NETBALL | VOLLEYBALL | CYCLING | null",
  "tournamentSlug": "string | null",
  "venue":          "string?",
  "scheduledAt":    "ISO8601?",
  "organizer":      "string?",
  "description":    "string?"
}

Response 201: OtherEvent
```

### `PUT /other-events/:slug`

Same request shape as `POST`. Treat as a full replacement of the other-event document — the client always sends the complete state.

```jsonc
Response 200: OtherEvent
Response 404: { "error": "Other event not found" }
```

### `DELETE /other-events/:slug`

```jsonc
Response 204
```

## Error format

All non-2xx responses use the project-wide error shape:

```json
{ "error": "human-readable message", "code": "OPTIONAL_MACHINE_CODE" }
```
