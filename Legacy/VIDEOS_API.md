# Videos API Specification

Base URL: `{API_BASE_URI}` (configured via environment).

All endpoints expect/return `application/json`. Routes are reachable both with and without the `/api` prefix unless explicitly noted (e.g. `/videos` and `/api/videos` are equivalent).

This document covers the full Videos surface — CRUD plus the per-resource upload alias. The media-storage / presign contract used by `POST /videos/upload` is documented once in [`MEDIA_API.md`](./MEDIA_API.md) and is shared with the other per-resource upload aliases.

---

## Concepts

### Slug

Every video is identified by a URL-friendly `slug` (not a numeric id). The backend generates the slug, likely from `caption` (or `title` when present). The slug MUST be returned on every GET response and is used as the path parameter on `GET /videos/:slug`, `PUT /videos/:slug`, and `DELETE /videos/:slug`.

### `source` — YouTube ID vs hosted file

`source` is the single field that points to the actual media. It accepts two distinct kinds of value:

1. **YouTube video ID** (e.g. `"dQw4w9WgXcQ"`) — pasted by the admin from a YouTube URL. The frontend builds the embed URL on the fly.
2. **Hosted media URL** (e.g. `"https://cdn.example.com/scoreboard/videos/videos/2026/05/<uuid>.mp4"`) — returned by `POST /videos/upload` after the browser PUTs an mp4/webm/etc. file directly to S3. The client stores the returned `url` here.

The backend SHOULD distinguish the two by shape (a bare 11-char id vs a full `https://` URL) — both are valid and the field is not split into two columns.

### Enums

| Field    | Values                                                                                                                                  | Default     |
|----------|-----------------------------------------------------------------------------------------------------------------------------------------|-------------|
| `sport`  | `CRICKET \| RUGBY \| SOCCER \| BADMINTON \| BOWLING \| MOTOR_SPORTS \| BASKETBALL \| NETBALL \| VOLLEYBALL \| CYCLING`                   | required    |
| `status` | `ACTIVE \| HIDDEN`                                                                                                                      | `ACTIVE`    |

`HIDDEN` videos MUST be excluded from any public-facing list. They remain reachable by `slug` for admin preview.

### Authentication

- CRUD routes (`GET / POST / PUT / DELETE`) are **not** protected in code today, matching the rest of the resource spec.
- `POST /videos/upload` IS protected — send `x-access-token: <JWT>`. This mirrors all other `/<resource>/upload` aliases.

## Schema

### `Video`

```jsonc
{
  "slug":      "string",                                                            // canonical id, generated by backend
  "caption":   "string",
  "title":     "string | null",
  "sport":     "CRICKET | RUGBY | SOCCER | BADMINTON | BOWLING | MOTOR_SPORTS | BASKETBALL | NETBALL | VOLLEYBALL | CYCLING",
  "date":      "ISO8601",
  "source":    "string",                                                            // YouTube video ID OR hosted media URL — see "source" above
  "status":    "ACTIVE | HIDDEN",
  "createdAt": "ISO8601",
  "updatedAt": "ISO8601"
}
```

## Endpoints

### `GET /videos`

```jsonc
Response 200: Video[]
```

### `GET /videos/:slug`

```jsonc
Response 200: Video
Response 404: { "error": "Video not found" }
```

### `POST /videos`

```jsonc
Request:
{
  "caption": "string",                                                              // required
  "sport":   "CRICKET | RUGBY | SOCCER | BADMINTON | BOWLING | MOTOR_SPORTS | BASKETBALL | NETBALL | VOLLEYBALL | CYCLING", // required
  "date":    "ISO8601",                                                             // required
  "source":  "string (YouTube video ID or hosted media URL)",                       // required
  "status":  "ACTIVE | HIDDEN",                                                     // required, default ACTIVE
  "title":   "string?"
}

Response 201: Video
```

### `PUT /videos/:slug`

Same request shape as `POST`. Treat as a full replacement of the video document — the client always sends the complete state.

```jsonc
Response 200: Video
Response 404: { "error": "Video not found" }
```

### `DELETE /videos/:slug`

```jsonc
Response 204
```

## Hosted-file upload

### `POST /videos/upload`

Protected. Returns presigned S3 upload details. Same response shape as `POST /media/presign` with the folder hard-coded to `videos` — the client SHOULD omit `folder` when calling the alias; the backend MUST ignore any client-supplied `folder` value and use `videos`.

Used when the admin uploads a hosted video file (mp4 / webm / etc.) instead of pasting a YouTube URL. Once S3 returns `2xx` on the `PUT`, the client stores the returned `url` in `Video.source` and submits the `POST /videos` or `PUT /videos/:slug` as usual.

See [`MEDIA_API.md`](./MEDIA_API.md) for:
- the canonical request / response payload,
- the browser → S3 PUT flow (the API server never sees the file bytes),
- backend signing responsibilities (auth, key generation, size + content-type pinning, short expiry),
- and S3 bucket / CORS requirements.

Note that the `videos` folder MUST accept both `image/*` and `video/*` content types (per `MEDIA_API.md`), with a higher size cap than image-only folders.

All non-2xx responses use the project-wide error shape:

