# Tournaments API Specification

Base URL: `{API_BASE_URI}` (configured via environment).

All endpoints expect/return `application/json`. Routes are reachable both with and without the `/api` prefix unless explicitly noted (e.g. `/tournaments` and `/api/tournaments` are equivalent).

This document covers the full Tournaments surface — CRUD plus the per-resource upload alias. The media-storage / presign contract used by `POST /tournaments/upload` is documented once in [`MEDIA_API.md`](./MEDIA_API.md) and is shared with the other per-resource upload aliases.

---

## Concepts

### Slug

Every tournament is identified by a URL-friendly `slug` (not a numeric id). The backend generates the slug, likely from `name`. The slug MUST be returned on every GET response and is used as the path parameter on `GET /tournaments/:slug`, `PUT /tournaments/:slug`, and `DELETE /tournaments/:slug`.

`GET /tournaments` MUST include `slug` and `name` on every item — both are consumed by the match/event creation dropdowns in the admin UI.

### Enums

| Field           | Values                                                                                                                                 | Default      |
|-----------------|----------------------------------------------------------------------------------------------------------------------------------------|--------------|
| `sport`         | `CRICKET \| RUGBY \| SOCCER \| BADMINTON \| BOWLING \| MOTOR_SPORTS \| BASKETBALL \| NETBALL \| VOLLEYBALL \| CYCLING`                  | required     |
| `format`        | `LEAGUE \| KNOCKOUT \| GROUP_STAGE \| ROUND_ROBIN`                                                                                     | required     |
| `status`        | `UPCOMING \| ONGOING \| COMPLETED \| CANCELLED`                                                                                        | `UPCOMING`   |
| `cricketFormat` | `T20 \| OD \| T10 \| TEST \| HUNDRED \| null` — only meaningful when `sport = CRICKET`; inherited by linked matches on write (see below) | `null`       |

### Relationship to other resources

- **Matches** (see [`MATCHES_API.md`](./MATCHES_API.md)) reference tournaments via the `tournamentSlug` field — a FK to `Tournament.slug`, or `null` for independent / friendly matches.
- **Other Events** carry a `tournamentSlug` that is likewise a FK to `Tournament.slug` (or `null` when the event is not tied to a tournament).

Renaming a `Tournament` does NOT change its `slug`, so referencing rows stay valid. Deleting a tournament does NOT cascade — the backend SHOULD either reject the delete while referencing matches/events exist, or null out `tournamentSlug` on the referencing rows out-of-band.

### Cricket format inheritance

When `sport = CRICKET`, the tournament MAY carry a `cricketFormat` (`T20 | OD | TEST`). On `POST /matches` and `PUT /matches/:slug`, if the incoming match has a `tournamentSlug` and the request's `details.cricketFormat` is missing or `null`, the backend MUST copy the tournament's `cricketFormat` onto the stored match document. The inherited value is persisted on the match — subsequent reads see a self-contained match doc and are not affected by later changes to the tournament's `cricketFormat`. A non-null `details.cricketFormat` in the request always wins over the tournament value.

### Authentication

- CRUD routes (`GET / POST / PUT / DELETE`) are **not** protected in code today, matching the rest of the resource spec.
- `POST /tournaments/upload` IS protected — send `x-access-token: <JWT>`. This mirrors all other `/<resource>/upload` aliases.

## Schema

### `Tournament`

```jsonc
{
  "slug":          "string",                                                     // canonical id, generated by backend
  "name":          "string",
  "sport":         "CRICKET | RUGBY | SOCCER | BADMINTON | BOWLING | MOTOR_SPORTS | BASKETBALL | NETBALL | VOLLEYBALL | CYCLING",
  "format":        "LEAGUE | KNOCKOUT | GROUP_STAGE | ROUND_ROBIN",
  "status":        "UPCOMING | ONGOING | COMPLETED | CANCELLED",
  "cricketFormat": "T20 | OD | T10 | TEST | HUNDRED | null",                    // only meaningful when sport = CRICKET; inherited by linked matches on write
  "description":   "string | null",
  "startDate":     "ISO8601 | null",
  "endDate":       "ISO8601 | null",
  "location":      "string | null",
  "logoUrl":       "string | null",                                              // URL returned from /tournaments/upload (CDN / public URL)
  "createdAt":     "ISO8601",
  "updatedAt":     "ISO8601"
}
```

## Endpoints

### `GET /tournaments`

```jsonc
Response 200: Tournament[]   // every item MUST include slug + name (used in match/event dropdowns)
```

### `GET /tournaments/:slug`

```jsonc
Response 200: Tournament
Response 404: { "error": "Tournament not found" }
```

### `POST /tournaments`

```jsonc
Request:
{
  "name":          "string",                                                     // required
  "sport":         "CRICKET | RUGBY | SOCCER | BADMINTON | BOWLING | MOTOR_SPORTS | BASKETBALL | NETBALL | VOLLEYBALL | CYCLING", // required
  "format":        "LEAGUE | KNOCKOUT | GROUP_STAGE | ROUND_ROBIN",              // required
  "status":        "UPCOMING | ONGOING | COMPLETED | CANCELLED",                 // required, default UPCOMING
  "cricketFormat": "T20 | OD | T10 | TEST | HUNDRED | null",                    // optional; ignored unless sport = CRICKET
  "description":   "string?",
  "startDate":     "ISO8601?",
  "endDate":       "ISO8601?",
  "location":      "string?",
  "logoUrl":       "string?"                                                     // set this to the `url` returned by /tournaments/upload
}

Response 201: Tournament
```

### `PUT /tournaments/:slug`

Same request shape as `POST`. Treat as a full replacement of the tournament document — the client always sends the complete state.

```jsonc
Response 200: Tournament
Response 404: { "error": "Tournament not found" }
```

### `DELETE /tournaments/:slug`

```jsonc
Response 204
```

## Logo upload

### `POST /tournaments/upload`

Protected. Returns presigned S3 upload details. Same response shape as `POST /media/presign` with the folder hard-coded to `tournaments` — the client SHOULD omit `folder` when calling the alias; the backend MUST ignore any client-supplied `folder` value and use `tournaments`.

See [`MEDIA_API.md`](./MEDIA_API.md) for:
- the canonical request / response payload,
- the browser → S3 PUT flow (the API server never sees the file bytes),
- backend signing responsibilities (auth, key generation, size + content-type pinning, short expiry),
- and S3 bucket / CORS requirements.

Once S3 returns `2xx` on the `PUT`, the client stores the returned `url` in `Tournament.logoUrl` and submits the `POST /tournaments` or `PUT /tournaments/:slug` as usual.

## Error format

All non-2xx responses use the project-wide error shape:

```json
{ "error": "human-readable message", "code": "OPTIONAL_MACHINE_CODE" }
```
