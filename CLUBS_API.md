# Clubs API Specification

Base URL: `{API_BASE_URI}` (configured via environment).

All endpoints expect/return `application/json`. Routes are reachable both with and without the `/api` prefix unless explicitly noted (e.g. `/clubs` and `/api/clubs` are equivalent).

This document covers the full Clubs surface — CRUD plus the per-resource upload alias. The media-storage / presign contract used by `POST /clubs/upload` is documented once in [`MEDIA_API.md`](./MEDIA_API.md) and is shared with the other per-resource upload aliases.

---

## Concepts

### Slug

Every club is identified by a URL-friendly `slug` (not a numeric id). The backend generates the slug, likely from `name`. The slug MUST be returned on every GET response and is used as the path parameter on `GET /clubs/:slug`, `PUT /clubs/:slug`, and `DELETE /clubs/:slug`.

### Enums

| Field          | Values                                                                                                                                  | Default      |
|----------------|-----------------------------------------------------------------------------------------------------------------------------------------|--------------|
| `type`         | `GRASSROOTS \| SEMI_PRO \| PROFESSIONAL`                                                                                                | required     |
| `primarySport` | `CRICKET \| RUGBY \| SOCCER \| BASKETBALL \| NETBALL \| VOLLEYBALL \| BADMINTON \| CYCLING \| BOWLING \| MOTOR_SPORTS \| OTHER`          | required     |
| `status`       | `ACTIVE \| INACTIVE \| SUSPENDED`                                                                                                       | `ACTIVE`     |

### Authentication

- CRUD routes (`GET / POST / PUT / DELETE`) are **not** protected in code today, matching the rest of the resource spec.
- `POST /clubs/upload` IS protected — send `x-access-token: <JWT>`. This mirrors all other `/<resource>/upload` aliases.

## Schema

### `Club`

```jsonc
{
  "slug":           "string",                              // canonical id, generated by backend
  "name":           "string",
  "type":           "GRASSROOTS | SEMI_PRO | PROFESSIONAL",
  "primarySport":   "CRICKET | RUGBY | SOCCER | BASKETBALL | NETBALL | VOLLEYBALL | BADMINTON | CYCLING | BOWLING | MOTOR_SPORTS | OTHER",
  "status":         "ACTIVE | INACTIVE | SUSPENDED",
  "description":    "string | null",
  "city":           "string | null",
  "country":        "string | null",
  "foundedYear":    "number | null",
  "website":        "string | null",
  "email":          "string | null",
  "logoUrl":        "string | null",                       // URL returned from /clubs/upload (CDN / public URL)

  "secondaryColor": "string",                              // hex, default "#ffffff"
  "createdAt":      "ISO8601",
  "updatedAt":      "ISO8601"
}
```

## Endpoints

### `GET /clubs`

```jsonc
Response 200: Club[]
```

### `GET /clubs/:slug`

```jsonc
Response 200: Club
Response 404: { "error": "Club not found" }
```

### `POST /clubs`

```jsonc
Request:
{
  "name":           "string",                                                    // required
  "type":           "GRASSROOTS | SEMI_PRO | PROFESSIONAL",                      // required
  "primarySport":   "CRICKET | RUGBY | SOCCER | BASKETBALL | NETBALL | VOLLEYBALL | BADMINTON | CYCLING | BOWLING | MOTOR_SPORTS | OTHER", // required
  "status":         "ACTIVE | INACTIVE | SUSPENDED",                             // required, default ACTIVE
  "description":    "string?",
  "city":           "string?",
  "country":        "string?",
  "foundedYear":    "number?",
  "website":        "string?",
  "email":          "string?",
  "logoUrl":        "string?",                                                   // set this to the `url` returned by /clubs/upload
  "primaryColor":   "string? (hex, default #1976d2)",
  "secondaryColor": "string? (hex, default #ffffff)"
}

Response 201: Club
```

### `PUT /clubs/:slug`

Same request shape as `POST`. Treat as a full replacement of the club document — the client always sends the complete state.

```jsonc
Response 200: Club
Response 404: { "error": "Club not found" }
```

### `DELETE /clubs/:slug`

```jsonc
Response 204
```

## Logo upload

### `POST /clubs/upload`

Protected. Returns presigned S3 upload details. Same response shape as `POST /media/presign` with the folder hard-coded to `clubs` — the client SHOULD omit `folder` when calling the alias; the backend MUST ignore any client-supplied `folder` value and use `clubs`.

See [`MEDIA_API.md`](./MEDIA_API.md) for:
- the canonical request / response payload,
- the browser → S3 PUT flow (the API server never sees the file bytes),
- backend signing responsibilities (auth, key generation, size + content-type pinning, short expiry),
- and S3 bucket / CORS requirements.

Once S3 returns `2xx` on the `PUT`, the client stores the returned `url` in `Club.logoUrl` and submits the `POST /clubs` or `PUT /clubs/:slug` as usual.

---

## Error format

All non-2xx responses use the project-wide error shape:

```json
{ "error": "human-readable message", "code": "OPTIONAL_MACHINE_CODE" }
```
